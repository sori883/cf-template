# Planコマンド

このコマンドは**planner**エージェントを呼び出し、コード作成前に包括的な実装計画を作成する。

## このコマンドの機能

1. **要件の再確認** - 構築すべきものを明確化
2. **リスクの特定** - 潜在的な問題やブロッカーを洗い出す
3. **ステップ計画の作成** - 実装をフェーズに分解
4. **確認を待つ** - 続行前にユーザーの承認を必ず得る

## 使用タイミング

以下の場合に `/plan` を使用:
- 新機能の開発開始時
- 重要なアーキテクチャ変更時
- 複雑なリファクタリング作業時
- 複数のファイル/コンポーネントに影響がある場合
- 要件が不明確または曖昧な場合

## 動作の仕組み

plannerエージェントが以下を実行:

1. **リクエストを分析** し、要件を明確な言葉で再確認
2. **フェーズに分解** し、具体的で実行可能なステップを提示
3. **依存関係を特定** コンポーネント間の関連を把握
4. **リスクを評価** 潜在的なブロッカーを分析
5. **複雑度を見積もり** （高/中/低）
6. **計画を提示** し、明示的な確認を待つ

## 使用例（言語/スタック非依存）

```
ユーザー: /plan ドメインイベント発生時にリアルタイム通知を追加したい

エージェント（planner）:
# 実装計画: リアルタイム通知

## 要件の再確認
- 関心を登録したユーザーへドメインイベント発生時に通知を送信
- 複数の通知チャネルをサポート（アプリ内、メール、Webhook）
- 通知が確実に配信されることを保証（at-least-once）
- イベント詳細とユーザー固有のコンテキストを含める

## 実装フェーズ

### フェーズ1: データモデル
- 通知エンティティを追加（フィールド: id, user_id, source_id, type, status, created_at）
- ユーザー通知設定エンティティをチャネル設定用に追加
- 検索性能向上のため user_id と source_id にインデックスを作成

### フェーズ2: 通知サービス
- 通知サービスモジュールを作成（言語/フレームワークに対応）
- メッセージキューを使用した非同期通知配信を実装
- 配信失敗時のリトライロジック（指数バックオフ）を追加
- 通知テンプレートを作成

### フェーズ3: 統合ポイント
- ドメインイベント発火点にフック
- 関心登録された全ユーザーをクエリ
- 各ユーザーへの通知をキューに追加

### フェーズ4: クライアント側コンポーネント
- 通知一覧 UI コンポーネントを作成
- リアルタイム更新を実装（WebSocket / SSE / プッシュ）
- 通知設定 UI を追加

## 依存関係
- メッセージキュー（プロジェクトの選定に従う）
- メール送信サービス
- リアルタイム配信機構（WebSocket/SSE 等）

## リスク
- HIGH: メール到達率（SPF/DKIM が必要）
- MEDIUM: 大量ユーザーへの fan-out 性能
- MEDIUM: 高頻度イベントでの通知スパム
- LOW: リアルタイム接続のオーバーヘッド

## 推定複雑度: MEDIUM
- バックエンド: 4-6時間
- フロントエンド/クライアント: 3-4時間
- テスト: 2-3時間
- 合計: 9-13時間

**確認待ち**: この計画で進めますか？（yes/no/modify）
```

例の各要素は具体的なフレームワーク/サービス名にバインドしない。実装フェーズでは、プロジェクトに既に採用されているスタックを優先して選定する。

## 重要な注意事項

**CRITICAL**: plannerエージェントは、"yes"や"proceed"などの肯定的な応答で明示的に計画を確認するまで、**一切コードを書きません**。

変更が必要な場合は以下のように応答:
- "modify: [変更内容]"
- "different approach: [代替案]"
- "フェーズ2をスキップしてフェーズ3を先にやって"

## 他のコマンドとの連携

計画作成後:
- `/tdd` でテスト駆動開発による実装
- `/verify` で検証
- `/code-review` で完成した実装のレビュー

## 関連エージェント

このコマンドは `planner` エージェントを呼び出す。`planner` エージェントが利用可能であることを前提とする（`agents/architect.agent.md` 等のリポジトリ提供エージェント、または `~/.claude/agents/planner.md` へのインストール）。エージェントが見つからない場合、本コマンドのプロンプト本文を直接 inline で実行する。
