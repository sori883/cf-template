# 戦略的コンパクトスキル

任意の自動圧縮に頼るのではなく、**ツールコール数のしきい値で手動 `/compact` を提案するフック**を設定する。

**仕組みの限界**: このフックはツールコール数のみを見るため、論理的な区切りを直接認識するわけではない。提案が出たタイミングで、ユーザーが「いま圧縮してよい区切りか」を判断する。判断の指針は本ドキュメント末尾の「ベストプラクティス」を参照。

## なぜ戦略的な圧縮が必要か？

自動圧縮は任意のタイミングで発火する:
- タスクの途中で発生し、重要なコンテキストを失うことが多い
- 論理的なタスク境界を認識しない
- 複雑なマルチステップ操作を中断する可能性がある

論理的な境界での戦略的圧縮:
- **調査後、実行前** - リサーチコンテキストを圧縮し、実装計画を保持
- **マイルストーン完了後** - 次のフェーズのためにリフレッシュ
- **大きなコンテキスト切り替え前** - 別のタスクの前に調査コンテキストをクリア

## 仕組み

`suggest-compact.sh` スクリプトはPreToolUse（Edit/Write）で実行され:

1. **ツールコール数を追跡** - セッション内のツール呼び出し回数をカウント
2. **閾値検出** - 設定可能な閾値（デフォルト: 50回）で提案
3. **定期的なリマインダー** - 閾値後、25回ごとにリマインド

## フック設定

`~/.claude/settings.json` に追加（既存 `hooks` キーがあれば**追記**）:

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "~/.claude/skills/strategic-compact/suggest-compact.sh"
      }]
    }]
  }
}
```

**matcher の書式**: Claude Code の `matcher` は**正規表現マッチ**で、ツール名に対して照合される。`"Edit|Write"` は「Edit または Write」を捕捉する。式構文（`tool == "Edit"`）ではない点に注意。

**インストール**: スキルがリポジトリ配置の場合、`~/.claude/skills/strategic-compact/` に symlink またはコピーが必要:
```bash
ln -s /path/to/repo/skills/strategic-compact ~/.claude/skills/strategic-compact
```
または settings.json の `command` に**絶対パス**を直書きする。

## 設定

環境変数:
- `COMPACT_THRESHOLD` - 最初の提案までのツールコール数（デフォルト: 50）

## ベストプラクティス

1. **計画後に圧縮** - 計画が確定したら、圧縮してリフレッシュ
2. **デバッグ後に圧縮** - エラー解決コンテキストをクリアしてから続行
3. **実装中は圧縮しない** - 関連する変更のコンテキストを保持
4. **提案を読む** - フックは*いつ*を教えてくれるが、*するかどうか*はあなたが決める
